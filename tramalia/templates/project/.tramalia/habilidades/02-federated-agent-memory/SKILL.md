# Federated agent memory

## Propósito
Usar la memoria federada del repo.

## Cuándo usar
al iniciar cualquier sesión de trabajo.

## Workflow
1. Leer AGENTS.md y seguir su orden de lectura (docs/ai/00, 01, tarea en curso).
2. Consultar docs/ai/06-intentos-fallidos.md antes de proponer soluciones.
3. Registrar decisiones técnicas nuevas en docs/ai/05 (ADR).

## Guardrails
- No ejecutar comandos destructivos sin confirmación.
- No leer ni exponer secretos.
- No modificar archivos fuera del alcance de la tarea.

## Evidencia esperada
Referencias explícitas a los docs leídos; ADR creado si hubo decisión.
